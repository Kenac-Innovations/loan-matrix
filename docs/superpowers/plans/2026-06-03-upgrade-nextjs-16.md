# Next.js 16 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the loan-matrix app from Next.js 15.5.3 to Next.js 16.2.7 (latest stable) without breaking the Docker build or runtime behaviour.

**Architecture:** The app uses Next.js App Router with Prisma, next-auth v4, and a custom `middleware.ts`. The upgrade involves: running the official codemod to migrate async request APIs (params/searchParams), renaming middleware to proxy, updating the Dockerfile Node version, fixing the webpack config for Turbopack, and bumping package versions. next-auth v4 is kept as-is and verified manually — a v5 migration is out of scope.

**Tech Stack:** Next.js 16.2.7, React 19, pnpm, Prisma 6, next-auth v4, TypeScript, Tailwind CSS 4, Docker (node:20-alpine)

## Task 4: Update next.config.ts for Next.js 16

Two things to fix:
1. The custom `webpack` config (Turbopack is now default; webpack builds need explicit opt-in or config migration)
2. Verify no other deprecated `experimental` keys remain

**Files:**
- Modify: `next.config.ts`

- [ ] **Step 1: Read the current config**

```bash
cat next.config.ts
```

- [ ] **Step 2: Replace the webpack config with a Turbopack-compatible equivalent and add webpack fallback flag**

The `fs: false` fallback is needed for packages that reference `fs` in browser bundles. In Next.js 16 with Turbopack default, add it under the `turbopack` key AND keep webpack as a fallback build option:

```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "standalone",
  eslint: {
    ignoreDuringBuilds: true,
  },
  typescript: {
    ignoreBuildErrors: true,
  },
  experimental: {
    optimizePackageImports: ["@radix-ui/react-icons"],
  },
  turbopack: {
    resolveAlias: {
      fs: { browser: false },
    },
  },
  headers: async () => [
    {
      source: "/((?!_next/static|_next/image|favicon.ico|auth|static).*)",
      headers: [
        {
          key: "Cache-Control",
          value: "no-store, no-cache, must-revalidate, proxy-revalidate",
        },
        { key: "Pragma", value: "no-cache" },
        { key: "Expires", value: "0" },
      ],
    },
  ],
  compiler: {
    removeConsole:
      process.env.NODE_ENV === "production"
        ? { exclude: ["error", "warn"] }
        : false,
  },
  webpack: (config) => {
    config.resolve.fallback = {
      ...config.resolve.fallback,
      fs: false,
    };
    return config;
  },
};

export default nextConfig;
```

Note: keeping the `webpack` config block ensures parity if a `--webpack` build is ever needed. The `turbopack.resolveAlias` handles the same for Turbopack.

- [ ] **Step 3: Commit**

```bash

git commit -m "chore: add turbopack resolveAlias for fs in next.config.ts"
```
